# RAG 시스템 개요

RAG(Retrieval-Augmented Generation)는 정보 검색과 생성 모델을 결합한 혁신적인 AI 기술입니다.

## 주요 구성 요소

### 1. 문서 처리
RAG 시스템은 먼저 문서를 처리합니다. 이 과정에서 다음 작업이 수행됩니다:
- 문서 로딩: PDF, TXT, JSON 등 다양한 형식 지원
- 청킹(Chunking): 문서를 작은 단위로 분할
- 임베딩(Embedding): 텍스트를 벡터로 변환

### 2. 벡터 저장소
변환된 벡터는 Qdrant와 같은 벡터 데이터베이스에 저장됩니다.
- 고속 검색 지원
- 유사도 기반 검색
- 하이브리드 검색 (Dense + Sparse)

### 3. 검색 단계
사용자 쿼리가 들어오면:
1. 쿼리를 벡터로 변환
2. 벡터 DB에서 유사한 문서 검색
3. 관련성 높은 문서 반환

### 4. 생성 단계
검색된 문서를 바탕으로:
- LLM에 컨텍스트 제공
- 정확한 답변 생성
- 환각(Hallucination) 감소

## 성능 평가

RAG 시스템의 성능은 다음 메트릭으로 평가됩니다:
- Precision@K: 검색된 상위 K개 중 관련 문서 비율
- Recall@K: 전체 관련 문서 중 검색된 비율
- NDCG: 순위를 고려한 평가 지표
- MRR: 첫 번째 관련 문서의 순위

## 파이프라인 구조

파이프라인은 RAG 설정과 데이터 소스를 결합합니다:
- NORMAL Pipeline: 실제 문서 검색용
- TEST Pipeline: 평가 데이터셋과 연결

## 결론

RAG는 현대 AI 시스템에서 핵심적인 역할을 하며, 
정확하고 신뢰할 수 있는 정보 제공이 가능합니다.
